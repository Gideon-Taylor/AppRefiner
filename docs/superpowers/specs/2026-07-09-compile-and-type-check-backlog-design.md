## 1. Purpose

Living reference for compiler-fidelity and type-checking work that AppRefiner is
**not** doing yet (or only partially). Each item is sized for one session or a
small multi-session slice. Update status in place as items ship; do not invent
new plans for already-listed work without amending this doc.

**How to use this doc in a session**

1. Pick the next item with status `Ready` in priority order (or the user’s pick).
2. If the item still has open design questions, resolve them **before** coding and
   record the decision under that item’s **Notes / decisions**.
3. Implement with tests in `PeopleCodeParser.SelfHosted.Tests` (compile checks) or
   type-system tests as appropriate; surface compile diagnostics via
   `CompileChecker` + `CompilerErrorsStyler` (no new styler toggles for true
   compile errors).
4. Mark the item `Done`, link the commit SHA, and append any follow-ups under
   **Deferred / discovered**.

## 3. Already shipping (baseline — do not re-implement)

### Compile checks (`CompileChecker` / `DiagnosticCode`)

Missing semicolon, redeclared / undefined variable, class name mismatch, invalid
app class, unimported / ambiguous class, invalid member access (existence),
undeclared function, missing method implementation, missing constructor,
invalid break/continue, not-all-paths-return, missing return value (bare
`Return;` in value routine), unexpected return value (procedure/setter),
duplicate class/interface members, missing property get/set body, assignment to
constant, `%This` outside app class, missing `%Super` in subclass constructor,
inaccessible private/protected member.

### Type checker (`TypeCheckerVisitor` + inference)

Assignment compatibility, local declaration-with-init, function / `Create` arg
validation (`FunctionCallValidator`), discarded non-optional return values,
return-expression vs declared return type, logical expression required on
If/While/Repeat-Until. Inference records binary arithmetic / logical operator
errors, array subscript on non-array, non-numeric array index, and unary `Not` /
negate operand type errors on the node.

### Parse-level (not `ICompileCheck`)

Function parameters marked `OUT`; parameter list on `DECLARE FUNCTION …
PEOPLECODE`; orphan method/property implementations; instance/constant outside
private section.

### Intentional non-compile surfaces

| Surface | Why not compile-check |
|---|---|
| Unimplemented abstract / interface members | Advisory styler only; does not trip “Compiler errors” (code 13 reserved) |
| Wrong exception variable | Hygiene styler |
| Unused imports / variables | Style / cleanup |

## 5. Accepted work — compile checks (`ICompileCheck`)

Prefer pure AST / `NotRequired` when possible so offline editing still works.
Append new `DiagnosticCode` values only (never renumber). Register in
`CompileChecker.CreateChecks()`.

### C1 — Unexpected return value (procedure / no return type)

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P1 (next wave) |
| **Diagnostic** | `UnexpectedReturnValue` (= 18) |
| **Requirement** | `NotRequired` |

**Behavior:** Report when `Return <expr>;` appears inside a routine that has **no**
declared return type:

- Function with `ReturnType == null`
- Method (impl / declaration) with no return type
- Property **setter** (never returns a value)
- Do **not** flag property getters or value-returning methods/functions

**Message:** `Return value is not allowed here; {kind} '{name}' does not declare a return type.`

**Span:** The return expression.

**Tests:** `UnexpectedReturnValueCheckTests` — procedure function/method; setter;
value-returning OK; no overlap with `MissingReturnValue`.

**Notes / decisions:** Mirror of `MissingReturnValueCheck`. Message emphasizes
“does not declare a return type” rather than C#-style “void”. Implemented as
`UnexpectedReturnValueCheck`.

---

### C2 — Duplicate class / interface members

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P1 |
| **Diagnostic** | `DuplicateMember` (= 19) |
| **Requirement** | `NotRequired` |

**Behavior:** Within a single `AppClassNode` (class or interface) header, flag
duplicate names (case-insensitive) among:

- Methods (including constructors — still unique by name)
- Properties
- Instance variables
- Constants

**Span:** Second (and subsequent) declarations; message mentions earlier kind.

**Tests:** `DuplicateMemberCheckTests` — two methods; two properties; two
instances; method+property same name; interface duplicates; distinct names OK.

**Notes / decisions:** Same name illegal **regardless of kind** (method vs
property, etc.). Names compared as declared (instance/`&x` vs method `x` do not
collide). Instance/constant members only parse in `private` section.

### C3 — Property Get/Set completeness

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P1 |
| **Diagnostic** | `MissingPropertyAccessor` (= 20) |
| **Requirement** | `NotRequired` |

**Behavior:**

- Property declared with **Get** and no getter body → error on property / name
- Property declared with **Set** and no setter body → error
- Abstract properties exempt (no body expected), same spirit as
  `MissingMethodImplementationCheck` + abstract methods
- Orphan getter/setter without a property declaration is already a **parse**
  error — do not double-report if avoidable
- Interfaces skipped (no bodies expected)
- `readonly` / bare properties without `get`/`set` do not require accessors

**Tests:** `MissingPropertyAccessorCheckTests` — Get-only missing; Get+Set missing
one/both; abstract OK; fully implemented OK; readonly OK.

**Notes / decisions:** Implemented as `MissingPropertyAccessorCheck`.

### C4 — Constructor missing base (`%Super`) assignment

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P2 |
| **Diagnostic** | `MissingSuperCall` (= 23) |
| **Requirement** | `NotRequired` for the body walk (AST only); base presence is pure AST (`BaseType != null`). No resolver needed for *this* check. |

**PeopleCode rules (product owner, 2026-07-09):**

1. **If base has a parameterized constructor → subclass must declare a
   constructor.** Already implemented as `MissingConstructorCheck`
   (`Optional`, uses resolver + `Parameters.Count > 0`).
2. **If subclass defines a constructor and has a base class → constructor body
   must set `%Super`.** Independent of whether the base ctor is parameterized.
   Rationale: the compiler’s auto-generated constructor (which would set
   `%Super`) is suppressed once you write your own; you own the chain.
3. **Presence only:** `%Super` must be assigned **somewhere** in the constructor
   body (whole-body walk). Not path-sensitive; not required to be first
   statement. (Most app class ctors are small; all-paths would be nice later
   but is out of scope for v1.)

**Call form:** Assignment, not a call:

```peoplecode
%Super = create BaseClass(...);
```

Detect: `AssignmentNode` whose target is identifier `%Super`
(`IdentifierType.Super` or name equals `%Super`, case-insensitive). Prefer also
seeing RHS as `ObjectCreationNode` (`create …`) for a clean match; if product
later confirms other RHS shapes are legal, loosen then.

**When to run:**

- Class has `BaseType` (extends something)
- Class has a constructor method (method name equals class name) with a body
- Body does **not** contain a `%Super = …` assignment → error

**Do not require:** base parameterized, resolver, or create-arg validation (Create
arg typing stays with existing type checker).

**Span:** Constructor method name / header (declaration and/or implementation).
**Message sketch:** `Constructor for '{Class}' must set %Super to initialize the base class.`

**Notes / decisions:** Locked 2026-07-09. Complements MissingConstructorCheck;
does not replace it. No all-paths analysis in v1.

### C5 — Assignment to constant

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P1 |
| **Diagnostic** | `AssignmentToConstant` (= 21) |
| **Requirement** | `NotRequired` (uses scope / variable registry) |

**Behavior:** On assignment where the target is an `IdentifierNode`
(`UserVariable`) that resolves to `VariableKind.Constant`, report an error.
Covered: class constants and program-level constants. Reads of constants are OK.
`ClassConstantNode` paths are not l-values (`IsLValue == false`).

**Span:** Assignment target identifier.

**Tests:** `AssignmentToConstantCheckTests` — class constant write; program
constant write; local assign OK; read OK.

**Notes / decisions:** Uses registry kind (`VariableKind.Constant`), not name
heuristics. Implemented as `AssignmentToConstantCheck`.

### C6 — `%This` outside application class

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P2 |
| **Diagnostic** | `ThisOutsideClass` (= 22) |
| **Requirement** | `NotRequired` |

**Behavior:** Flag uses of `%This` (`IdentifierType.SystemVariable`) when
`Program.AppClass == null` (record FieldFormula, function libraries, plain
programs). Class **and** interface programs allow `%This` (both set `AppClass`).

**Span:** The `%This` identifier node.

**Tests:** `ThisOutsideClassCheckTests` — app class method OK; function /
record-style program error.

**Notes / decisions:** Message: `%This is only valid inside an application class.`
Implemented as `ThisOutsideClassCheck`.

## 6. Accepted work — type checker (`TypeCheckerVisitor` / inference)

These emit type errors/warnings through the existing type-error →
`DiagnosticCode.TypeError` / `TypeWarning` path (no new compile-check units
unless a pure-AST rule is cleaner).

### T1 — Array subscript on non-array

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P1 |

**Behavior:** On `ArrayAccessNode`, if the base expression’s inferred type is
known and not array-like / indexable, record a type error.

**Caveats:** `Any` / `Unknown` stay silent.

**Notes / decisions:** Already present in `TypeInferenceVisitor.VisitArrayAccess`
before this backlog wave; covered by `ArrayIndexAndUnaryTypeCheckTests`.

---

### T2 — Array index type not numeric

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P1 |

**Behavior:** Each index expression must be Number/Integer (or Any/Unknown).
Non-numeric known types → type error on the **index** expression.

**Notes / decisions:** Implemented in `TypeInferenceVisitor.VisitArrayAccess` via
`CanBeArrayIndex`. Date/Time are **not** valid indices. Errors recorded at
inference-site (same home as binary ops / T1).

---

### T3 — Unary `Not` / negate operand types

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P2 |

**Behavior:** Mirror binary-operator validation in `TypeInferenceVisitor`:

- `Not` requires logical-capable operand (`CanPerformLogical` — Boolean /
  Any / Unknown)
- Unary negate requires Number/Integer (`CanPerformUnaryNegate`; not date/time)

**Notes / decisions:** Errors recorded at **inference-site** (same as binary
operators). Unary negate deliberately narrower than binary arithmetic (no
date/time).

---

### T4 — L-value required for assignment

| Field | Value |
|---|---|
| **Status** | `Deferred` |
| **Priority** | P2 |

**Behavior (intent):** Assignment target must be a writable location (variable,
property, field value, array element, etc.). Reject assigning into non-lvalues
(e.g. bare function call result, literal) when the real compiler does.

**Notes / decisions:** Deferred 2026-07-09 (product owner: skip for now).

**Design survey (for when resumed):**

- **Structural** invalid LHS is already enforced at parse time via
  `ExpressionNode.IsLValue` → `"Invalid assignment target"` (including
  shorthand `+=`/`-=`/`|=`).
- **Constants** already covered by **C5** (`AssignmentToConstantCheck`).
- **OUT / MustBeVariable** already uses `TypeInfo.IsAssignable` in
  `FunctionCallValidator` — do not fork a second lattice.
- Remaining value if revived: **semantic** writability only (Option B) —
  e.g. assign to get-only / `readonly` property; not dual structural re-check
  (avoids double diagnostics with parse).
- Open if resumed: get-only property writes; SelfMetadata-only vs Optional
  resolver; bare Super as LHS; message text.

### T5 — Logical expression required (`If` / `While` / …)

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P2 |

**Behavior:** Per compiler: **“Logical expression required”** for condition
positions:

- `If` condition
- `While` condition
- `Repeat` … `Until` condition

Known non-logical types should error; `Any`/`Unknown` stay silent. Align
“logical-capable” with binary `And`/`Or` and unary `Not` (`CanPerformLogical` /
Boolean only — no number/string truthiness). Comparisons already infer as
Boolean and pass.

**Out of scope for T5:**

- `For` bounds/step (numeric, not logical)
- `Evaluate` / `When` — **not** boolean conditions. Forms are `When <value>`
  (implied `=`) or `When <op> <value>`; the AST’s `WhenClause.Condition` is the
  RHS match value against the Evaluate subject, not an If-style logical expr.
  A future check might validate subject/RHS comparability; that is separate.

**Home:** `TypeCheckerVisitor` (`VisitIf` / `VisitWhile` / `VisitRepeat`);
message on the condition span; surface as existing `DiagnosticCode.TypeError`.

**Notes / decisions:** Locked 2026-07-09 with product owner: Evaluate/When
clarified; still excluded from T5. Everything else (forms, strict boolean,
message, TypeCheckerVisitor home) approved.

### T6 — Member access visibility (private / protected)

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P3 |
| **Diagnostic** | `InaccessibleMember` (= 24) |
| **Requirement** | `Optional` (same as InvalidMemberAccessCheck) |

**Behavior:** After existence succeeds in `InvalidMemberAccessCheck`, enforce:

| Visibility | Accessible from |
|---|---|
| Public | Anyone |
| Protected | Declaring class + subclasses (caller inherits from declarer) |
| Private | Declaring class only |

- **Sibling classes** cannot access each other's protected members.
- **Instance variables** are always private (builder must stamp `Private`;
  only legal in private section).
- Caller class = SelfMetadata / current app class program; non-class → public only.
- Private blocked even via `%Super`.
- Soft-skip when metadata/type unresolved (no false positives).
- Builtin members: all public (no visibility check).

**Home:** Same class as existence — `InvalidMemberAccessCheck`.

**Notes / decisions:** Locked 2026-07-09: (1) new InaccessibleMember (2) no
sibling protected (3) instance vars always private (4) same check class.

## 7. Analyzer polish (optional)

### A1 — `CompletionAnalyzer` Repeat-loop precision

| Field | Value |
|---|---|
| **Status** | `Done` |
| **Priority** | P3 |

**Intent:** Tighten `Normal` precision for Repeat loops so return-path /
exit-mode consumers get more accurate results.

**Behavior:** For/While still force `Normal` (zero iterations). Repeat does
**not** force `Normal` after absorbing Break/Continue — body always runs once.
If body is only abrupt (Return/Throw/Exit/Error), Repeat has no `Normal`.

**Notes / decisions:** Implemented 2026-07-09. `AnalyzeForOrWhile` vs
`AnalyzeRepeat`. Tests: CompletionAnalyzer Repeat cases + NotAllPathsReturn
return-only-inside-repeat clean / fallthrough-body reports.
