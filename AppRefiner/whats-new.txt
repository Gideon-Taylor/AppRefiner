VERSION 1.3.0
=============

*** PLEASE READ: RESTART APPLICATION DESIGNER ***
-------------------------------------------------

This release updates the AppRefiner integration that runs inside Application
Designer. After updating, close and reopen any Application Designer windows
that were already open — until you do, the new multi-instance and autocomplete
fixes below will not be active in those windows.


NEW FEATURES
------------

• Instances Tab - A new "Instances" tab lists every Application Designer
  session AppRefiner is tracking, showing the active session, database
  connection, PeopleTools version, open editor count, and whether the enhanced
  editor is loaded. Connect or disconnect a database, or bring a session to the
  front, per instance — no more guessing which Application Designer a button
  acts on.
• Oracle LDAP Connections - The Database Connect dialog now supports connecting
  to Oracle via LDAP, resolving the service descriptor automatically so no
  separate ldap.ora configuration is required.
• tnsnames.ora IFILE Support - AppRefiner now follows IFILE includes when
  reading your tnsnames.ora, so aliases defined in included files appear in the
  connection list.
• Faster Connect Dialog - Database connection lists now load in the background,
  so the dialog opens immediately instead of pausing while it reads
  configuration.
• Undeclared Function Detection - Calls to functions that aren't built in,
  declared, or defined above their first use are underlined as you type —
  including the classic "defined below its first use" mistake. Press Ctrl+.
  on the underline to import the function from your function cache (works
  even without a database connection), search for it in the Declare Function
  dialog, or move a below-defined function above its caller automatically.
• Function Name Autocomplete - Press Ctrl+Space while typing a plain identifier
  (no & or % prefix) to see the functions you can call there: functions defined
  or declared in the current program — honoring PeopleCode's rule that an
  implementation must appear above its use — plus every builtin function. Pick
  one and type ( to get its signature call tip.
• Capture-Result Quick Fix - The "return/expression values must be assigned
  to a variable" type errors now offer a Ctrl+. fix that captures the value
  in a new local variable — typed from what AppRefiner inferred, named &_
  (or the next free underscore name in that scope).
• Redesigned Snapshot History - Restoring a previous version is now a
  side-by-side experience: pick a snapshot on the left and its diff against
  your current code renders instantly on the right — no more opening and
  closing View Diff for each guess. Rows show at-a-glance change counts
  (+12 −3), the preview offers Changes-only, Full-file, and Snapshot-content
  views plus a comparison-direction toggle (all remembered between sessions),
  and Copy Original / Copy Diff buttons put the old code or a unified diff on
  the clipboard. Applying a revert automatically snapshots any unsaved edits
  first so nothing is ever lost, and the dialog is resizable.


RELIABILITY & MULTI-INSTANCE FIXES
----------------------------------

This release includes a large overhaul of how AppRefiner communicates with
Application Designer, resolving a class of intermittent issues — especially
when more than one Application Designer is open.

• Editor Features No Longer Fade Away - Fixed a long-standing timing issue
  where, if Application Designer was started while AppRefiner was already
  running, editors opened a few minutes into the session could silently lose
  autocomplete, the AppRefiner button, and the minimap until both programs
  were restarted. Editors now stay fully functional for the life of the
  session.
• Instant Editor Setup - Editors are now wired up the moment they are created
  instead of waiting for you to click into them, and the setup is verified and
  automatically retried if Application Designer wasn't ready yet. Code analysis
  visuals (inlay hints, error highlights) also appear as the editor opens
  rather than on first click. When AppRefiner starts while Application Designer
  already has editors open, those editors are initialized immediately.
• Enhanced Editor Without Restarting - If AppRefiner attaches to an Application
  Designer that already has editors open, the enhanced editor previously
  required a full Application Designer restart. Now it loads automatically as
  soon as you close all open editors — no restart, no losing your place.
• Correct Instance Routing - Autocomplete, tooltips, call tips, paste handling,
  and save tracking are now always applied to the editor that triggered them.
  Previously, switching quickly between editors or running several Application
  Designers could send a result to the wrong window.
• "Buffers Go Weird" Fixes - Eliminated the race conditions behind occasional
  wrong autocomplete values, garbled tooltips, and stale content — the reads
  and writes AppRefiner makes into Application Designer's memory are now
  isolated and synchronized.
• Auto-Connect Prompt Placement - When "Prompt for DB Connection" is enabled
  and multiple Application Designers are open at startup, the connect dialog
  now appears on the Application Designer you actually switch to, instead of
  opening invisibly behind other windows.
• Session Lifecycle - Closing an editor or exiting an Application Designer no
  longer leaves stale state behind. Reopening editors and relaunching
  Application Designer now work correctly without restarting AppRefiner.


BUG FIXES
---------

• Inlay Hints Survive Saving - Parameter name hints (and error highlights) no
  longer disappear when you save. Application Designer rewrites the editor
  contents during a save; AppRefiner now restores its visuals right after.
• Accurate Enhanced Editor Status - The Instances tab's "Enhanced" column now
  reflects whether the enhanced editor is actually loaded. Previously it always
  showed "No" on Tools 8.60 and always "Yes" on 8.61+, regardless of reality.
• Autocomplete Doubling - Accepting a suggestion after typing part of a name no
  longer duplicates the text (e.g. "&varvariable"), including in files that
  contain non-ASCII characters.
• Lingering Error Highlights - Parse and type-error squiggles shown while you
  type are now cleared automatically once the code becomes valid again — no
  more leftover highlights at the wrong positions that required a save or
  refresh to clear.
• Class Autocomplete Semicolon - A trailing ";" is only added when completing a
  class inside an import statement, not mid-expression.
• Find & Replace - "Replace All" now searches the correct term when used more
  than once in a session, and no longer inserts stray characters.
• Signature Updates - Added undocumented Json* methods, corrected SQLExec and
  Hash.SetValue() types, handled "@" references in built-in signatures, and
  capitalized all system variables.


PERFORMANCE
-----------

• Responsive Editing - Stylers now run asynchronously, keeping the editor
  responsive while analysis happens in the background.
• Fewer Cross-Process Reads - AppRefiner reads far less document data while you
  type, reusing cached parse results when the document hasn't changed.
• Member Access Caching - Member access validation is memoized, with automatic
  invalidation on save or when switching Application Designer sessions.


VERSION 1.2.1
=============

BUG FIXES
---------

• Autocomplete - Fixed autocomplete being broken in the 1.2.0 release.
• Array.ForEach() - Fixed generated code missing a semicolon at the end of the
  loop variable declaration.
• SQL Formatting on Paste - Fixed SQL auto-formatting not applying correctly
  when pasting into a string literal.

INTERNAL IMPROVEMENTS
---------------------

• Remote Buffer Handling - Overhauled how AppRefiner allocates memory in the
  Application Designer process. Buffers are now individually named or
  temporary rather than sharing one large allocation, improving reliability
  and reducing memory overhead.
• Performance - Line-text retrieval now uses a direct range pointer, reducing
  unnecessary cross-process reads.


VERSION 1.2.0
=============

NEW FEATURES
============

Enhanced Editor & Parameter Name Inlay Hints
----------------------------------------------
• Parameter Name Inlay Hints - Function parameter names displayed as inline hints at call sites,
  including inside object constructors. Requires the "Use Enhanced Editor" setting to be enabled.
• Enhanced Editor - Opt-in setting (General Settings > Feature Overrides > "Use Enhanced Editor")
  loads a modified Scintilla editor at runtime. Your Application Designer files are NOT modified —
  this is a runtime-only change. A restart of Application Designer is required.

Code Quality & Styling
-----------------------
• Invalid Member Access Styler - Highlights invalid member access such as calling a non-existent
  method or property on a typed object

Editor Enhancements
---------------------
• Toggle Line Comment - Comment or uncomment selected lines with Ctrl+/ using aligned block
  comment markers (<* ... *>)
• Document Minimap - A minimap overview of your code displayed alongside the editor, default state set via
  AppRefiner settings. Toggle-able via the AppRefiner button in the editor.
• Smart Paste - When pasting text into a string literal, special characters are automatically escaped
• AppRefiner Button - New in-editor button with a context menu offering quick access to the
  Command Palette, Toggle Minimap, and Toggle Parameter Names (if enabled).

Language Extensions
--------------------
• Array .Filter() - New language extension for filtering arrays
• %Response.HTMLLine() - New type extension that makes appending a break tag automatic


IMPROVEMENTS
============

• Go To Definition - Now works on chained member access expressions (e.g. thing.property.method())
• Declare Function - Improved behavior in various contexts
• Autocomplete Conflict Warning - Alert when both AppRefiner and App Designer autocompletions are
  enabled simultaneously
• Language Extensions - Support for implicit parameters and contextual usage info for transforms
• Plugin UX - Improved plugin management and fixed duplication issue with plugin
  commands/linters/refactors
• Dark Mode - Properly style /* */ block comments
• Improved highlighting in function call tips


BUG FIXES
=========

• Fix false reports of unused imports
• Fix issue where function call tip shows the wrong one
• Fix issue with string interpolation that prevents sequential interpolation blocks from parsing
  correctly
• Fix issue with using ":" as a fillup character in Tools 8.60
• Only check for call tips and interpolated strings in PeopleCode editors
