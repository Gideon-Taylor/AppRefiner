VERSION 1.3.0
=============

*** PLEASE READ: RESTART APPLICATION DESIGNER ***
-------------------------------------------------

This release updates the AppRefiner integration running inside Application
Designer. Close and reopen any Application Designer that was already open —
until you do, the multi-instance and autocomplete fixes below aren't active.


NEW FEATURES
------------

• Instances Tab - See every Application Designer session AppRefiner is
  tracking — database, Tools version, open editors — and connect, disconnect,
  or bring one to the front. No more guessing which window a button acts on.
• Oracle LDAP - Connect to Oracle via LDAP (contributed by NoodleSploder)
• tnsnames.ora IFILE Support - Aliases defined in IFILE-included files now
  appear in the connection list.
• Faster Connect Dialog - Connection lists load in the background, so the
  dialog opens instantly.
• Undeclared Function Detection - Functions that aren't built in, declared, or
  defined above their first use are underlined as you type. Press Ctrl+. to
  import the declaration from your function cache (no database needed), search
  for it, or move a below-defined function above its caller automatically.
• Function Name Autocomplete - Ctrl+Space on a plain identifier lists every
  function you can call there — defined, declared, and builtin — and typing (
  brings up its signature call tip.
• Capture-Result Quick Fix - "Value must be assigned to a variable" errors
  offer a Ctrl+. fix that captures the result into a new local variable,
  correctly typed and named for you.
• Redesigned Snapshot History - Pick a snapshot and instantly see its diff
  against your current code side by side, with per-row change counts and
  one-click copy of the old code or a unified diff. Reverting snapshots your
  unsaved edits first, so nothing is ever lost.
• Message Catalog - Hover any MsgGet-style call to see the message, severity,
  and explain text. Ctrl+Space inside the arguments opens a searchable browser
  that fills in the numbers and text for you — and helps you pick a valid free
  number when planning a new message.
• Extract Local Variable - Select any expression and pull it out into a new,
  correctly typed local declared just above the statement. The type is inferred
  even without a database connection, and when the expression appears more than
  once you can replace every occurrence at the same time.
• Inline Variable - Place the cursor on a local variable to fold its value back
  into each place it's used and remove the declaration. AppRefiner refuses when
  inlining would change behavior — a side-effecting initializer read more than
  once, or a value that changes before it's used — so the result always means
  the same thing.
• Convert If ↔ Evaluate - Turn an If/Else-If chain into an Evaluate statement,
  or an Evaluate back into an If, with one command that picks the direction from
  where your cursor sits. Break semantics, When-Other, and stacked-When "Or"
  conditions are all handled, and it refuses the cases that can't convert
  cleanly rather than changing your logic.
• Extract Function/Method - Select a run of statements and pull them into a new
  function — or a private method when you're in an App Class. AppRefiner figures
  out which variables need to be passed in, lets you pick one to return, and
  turns any additional outputs into "out" parameters, then replaces the
  selection with a call to the new routine. It refuses selections it can't move
  safely, such as those containing a Return.
• Statement Number Margin - Optionally repurpose Application Designer's
  line-number margin to display PeopleCode statement numbers instead. Locate the
  statement from a stack trace by scrolling to it, rather than using App
  Designer's Goto-by-statement dialog. Turn it on in General Settings.
• Unified Compiler Error Check - The fourteen separate checks that flagged code
  that won't compile now live under a single "Compiler errors" toggle, on by
  default — so you see everything that's wrong at once instead of enabling each
  one by hand. It covers syntax errors, type errors, missing semicolons,
  redeclared variables, undefined variables (in App Classes), class name
  mismatches, invalid application classes, unimported classes, ambiguous class
  references, invalid member access, undeclared functions, missing method
  implementations, missing constructors, and unimplemented abstract members.
  All of their quick fixes are preserved, and everything now runs in a single
  pass over your code. (Undefined-variable flagging outside App Classes remains
  its own optional style, where it's a code smell rather than a compile error.)


PARSER ENGINE OVERHAUL
----------------------

The PeopleCode parser behind linting, styling, and refactoring was deeply
audited this release (~50 fixes, now guarded by an automated test suite).

• Valid Code Parses Cleanly - Fixed false errors on legal PeopleCode: `&y-1`
  without spaces, `Not` with comparisons, keyword-named methods, mid-line REM,
  case-differing implementations, and more. If App Designer accepts it,
  AppRefiner should too.
• One Typo, One Error - Mistakes stay contained while you type: a typo or
  missing End-If no longer blanks styling for the rest of the class or
  swallows the code below it, and error messages point at the actual problem
  ("'End-If' has no matching 'If'") instead of flooding the list.
• No Crashes or Hangs - Input that could previously crash AppRefiner or pin a
  background thread at 100% CPU now just reports an error and moves on.
• More Accurate Analysis - Property getters, annotations, and Catch variables
  are now fully visible to linters, rename, and unused-variable detection, and
  quick fixes/refactors always edit exactly the text they should.


RELIABILITY & MULTI-INSTANCE FIXES
----------------------------------

A large overhaul of how AppRefiner talks to Application Designer, resolving a
class of intermittent issues — especially with multiple instances open.

• Editor Features No Longer Stop Working - Editors opened mid-session no
  longer silently lose autocomplete, the AppRefiner button, or the minimap —
  they stay fully functional for the life of the session.
• Instant Editor Setup - Editors are wired up the moment they open — hints and
  highlights included — instead of waiting for your first click. Editors
  already open when AppRefiner starts are picked up immediately.
• Enhanced Editor Without Restarting - Attaching to a running Application
  Designer no longer requires restarting it for the enhanced editor; it loads
  as soon as you close the open editors.
• Correct Instance Routing - Autocomplete, tooltips, call tips, and save
  tracking always land in the editor that triggered them — even when switching
  quickly or running several Application Designers.
• Wrong Values and Garbled Text - Eliminated the race conditions behind
  occasional wrong autocomplete values, garbled tooltips, and stale content.
• Auto-Connect Prompt Placement - The startup connect prompt appears on the
  Application Designer you actually switch to, instead of hiding behind other
  windows.
• Session Lifecycle - Closing editors or exiting an Application Designer no
  longer leaves stale state behind — reopening and relaunching just work.


BUG FIXES
---------

• Inlay Hints Survive Saving - Parameter name hints and error highlights no
  longer disappear when you save.
• Accurate Enhanced Editor Status - The Instances tab's "Enhanced" column now
  reflects whether the enhanced editor is actually loaded, instead of being
  decided by your Tools version.
• Autocomplete Doubling - Accepting a suggestion after typing part of a name no
  longer duplicates the text (e.g. "&varvariable"), including in files that
  contain non-ASCII characters.
• Lingering Error Highlights - Squiggles now clear automatically once the code
  is valid again — no more saving or refreshing to clean them up.
• Class Autocomplete Semicolon - A trailing ";" is only added when completing a
  class inside an import statement, not mid-expression.
• Find & Replace - "Replace All" now searches the correct term when used more
  than once in a session, and no longer inserts stray characters.
• Signature Updates - Added undocumented Json* methods, corrected SQLExec and
  Hash.SetValue() types, handled "@" references in built-in signatures, and
  capitalized all system variables.
• "Out" on Function Parameters - Marking a plain Function parameter with "Out"
  is now flagged as an error — Function parameters are always passed by
  reference, so "Out" only applies to App Class method parameters. This matches
  what the PeopleCode compiler rejects.
• Parameters on Declare Function ... PeopleCode - A "Declare Function ...
  PeopleCode" declaration that carries a parameter list is now flagged, since
  its parameters are implicit to the referenced record field event and aren't
  allowed here. Library (DLL) declarations with Ref/Value parameters are
  unaffected.
• New Class Members Recognized Immediately - Methods and properties you add to
  the class you're editing — via Extract Function/Method or by hand — are no
  longer flagged as unknown until you save and edit a second time. The member
  access check now sees the live, unsaved class.


PERFORMANCE
-----------

• Responsive Editing - Stylers now run asynchronously, keeping the editor
  responsive while analysis happens in the background.
• Fewer Cross-Process Reads - AppRefiner reads far less document data while you
  type, reusing cached parse results when the document hasn't changed.
• Member Access Caching - Member access validation is memoized, with automatic
  invalidation on save or when switching Application Designer sessions.


VERSION 1.2.1
=============

BUG FIXES
---------

• Autocomplete - Fixed autocomplete being broken in the 1.2.0 release.
• Array.ForEach() - Fixed generated code missing a semicolon at the end of the
  loop variable declaration.
• SQL Formatting on Paste - Fixed SQL auto-formatting not applying correctly
  when pasting into a string literal.

INTERNAL IMPROVEMENTS
---------------------

• Remote Buffer Handling - Overhauled how AppRefiner allocates memory in the
  Application Designer process. Buffers are now individually named or
  temporary rather than sharing one large allocation, improving reliability
  and reducing memory overhead.
• Performance - Line-text retrieval now uses a direct range pointer, reducing
  unnecessary cross-process reads.


VERSION 1.2.0
=============

NEW FEATURES
============

Enhanced Editor & Parameter Name Inlay Hints
----------------------------------------------
• Parameter Name Inlay Hints - Function parameter names displayed as inline hints at call sites,
  including inside object constructors. Requires the "Use Enhanced Editor" setting to be enabled.
• Enhanced Editor - Opt-in setting (General Settings > Feature Overrides > "Use Enhanced Editor")
  loads a modified Scintilla editor at runtime. Your Application Designer files are NOT modified —
  this is a runtime-only change. A restart of Application Designer is required.

Code Quality & Styling
-----------------------
• Invalid Member Access Styler - Highlights invalid member access such as calling a non-existent
  method or property on a typed object

Editor Enhancements
---------------------
• Toggle Line Comment - Comment or uncomment selected lines with Ctrl+/ using aligned block
  comment markers (<* ... *>)
• Document Minimap - A minimap overview of your code displayed alongside the editor, default state set via
  AppRefiner settings. Toggle-able via the AppRefiner button in the editor.
• Smart Paste - When pasting text into a string literal, special characters are automatically escaped
• AppRefiner Button - New in-editor button with a context menu offering quick access to the
  Command Palette, Toggle Minimap, and Toggle Parameter Names (if enabled).

Language Extensions
--------------------
• Array .Filter() - New language extension for filtering arrays
• %Response.HTMLLine() - New type extension that makes appending a break tag automatic


IMPROVEMENTS
============

• Go To Definition - Now works on chained member access expressions (e.g. thing.property.method())
• Declare Function - Improved behavior in various contexts
• Autocomplete Conflict Warning - Alert when both AppRefiner and App Designer autocompletions are
  enabled simultaneously
• Language Extensions - Support for implicit parameters and contextual usage info for transforms
• Plugin UX - Improved plugin management and fixed duplication issue with plugin
  commands/linters/refactors
• Dark Mode - Properly style /* */ block comments
• Improved highlighting in function call tips


BUG FIXES
=========

• Fix false reports of unused imports
• Fix issue where function call tip shows the wrong one
• Fix issue with string interpolation that prevents sequential interpolation blocks from parsing
  correctly
• Fix issue with using ":" as a fillup character in Tools 8.60
• Only check for call tips and interpolated strings in PeopleCode editors
